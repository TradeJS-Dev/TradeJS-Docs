---
title: Replay промптов из бэктестов
---

TradeJS умеет превращать AI-бэктесты в переиспользуемые датасеты для replay промптов.

AI-проверка в TradeJS не обязана жить только в live-режиме. Во время бэктеста можно записать replayable AI rows, а потом прогонять те же исторические сделки через обновлённую prompt-логику.

## Зачем это нужно

- Можно менять prompt behavior без полного повторного прогона рыночной симуляции.
- Можно сравнивать модели, провайдеров и quality thresholds на одном и том же историческом срезе.
- Можно заранее измерять ложные подтверждения и ложные отказы до изменения live AI gating.
- AI-оценка остаётся привязанной к реальным outcome сделок, а не к синтетическим примерам.

## Что TradeJS записывает во время бэктеста

Когда в бэктесте включен экспорт AI dataset, TradeJS пишет по каждой сделке строки с:

- идентификатором сигнала, `symbol`, `direction` и `timestamp`
- именем стратегии
- структурированным AI payload, по которому позже можно пересобрать strategy prompts
- фактическим `profit` сделки для оффлайн-оценки
- дополнительными test metadata для трассировки бэктеста

Сначала строки пишутся в chunk-файлы воркеров, потом объединяются в один replay dataset.

## Как работает prompt replay

1. Запускаете бэктест с включённым AI dataset export.
2. Объединяете chunk-файлы в один timestamped dataset.
3. Прогоняете этот dataset через текущую AI prompt-логику.
4. Сравниваете решения AI с фактическим результатом сделок до изменения live gating.

Во время replay TradeJS пересобирает пару промптов из сохранённого signal/payload контекста, поэтому изменения prompt-ов и strategy adapter можно проверять на одном и том же историческом срезе.

## Воспроизводимый CLI flow

```bash
npx @tradejs/cli backtest --config TrendLine:base --ai
npx @tradejs/cli ai-export
npx @tradejs/cli ai-train -n 50 --minQuality 4
```

Артефакты:

- `backtest --ai` пишет `data/ai/export/ai-dataset-<strategy>-chunk-<chunkId>.jsonl`
- `ai-export` объединяет их в `data/ai/export/ai-dataset-<strategy>-merged-<timestamp>.jsonl`
- `ai-train` по умолчанию воспроизводит строки из последнего merged-файла

## Что можно валидировать

- изменения промптов в strategy `aiAdapter`
- смену провайдера или модели
- настройку `minQuality`
- согласованность с исходным направлением стратегии
- поведение `tp / fp / tn / fn` для AI approval против реальной прибыльности
- локальные детерминированные эксперименты через `--localOnly`

## Почему это полезно до выхода в live

Prompt replay даёт безопасный цикл итерации:

1. Меняете prompt-логику или правила adapter в коде.
2. Повторно прогоняете тот же исторический набор строк.
3. Проверяете, улучшился ли tradeoff между покрытием и качеством.
4. Продвигаете новый gate только после внятного оффлайн-обоснования.

Так AI gating становится проверяемым, повторяемым и понятным для авторов стратегий.
